Merging conflicts

At some stage I run into merging conflicts. This was because I added a figure within an existing text bloc using the Authorea interface . Authorea then needs to splits the text bloc into two and adds a new  html file to the repository. 
  • When changing things on GitHub the changes are sometimes not seen online in the manuscript although in the Data folder the changes are already there. I suspect has to do with a cache that needs to be reloaded. But simply reload the html-page does usually not help. Why is that?
  • When you add a figure authorea saves the figure two times (one is labeled as the original). What is the use of it. When I change the figure in R should I overwrite both figure?
  • Sometimes it seems necessary to manually pull from or push to GitHub. This is rather easy just ad /git_functions at the end of the url.
